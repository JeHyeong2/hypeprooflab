# 반복 패턴 분석

## 🔄 자주 발생하는 작업 패턴들

### 1. **코드 리뷰 → 최적화** 패턴
**빈도**: 높음
**단계**:
1. Critic이 성능/구조 문제 지적
2. Developer가 구체적 코드 수정
3. 빌드 테스트로 검증
4. 결과 확인

**예시**:
- Critic: "애니메이션 과다로 성능 우려"
- Developer: scaleIn 변수를 인라인으로 변경
- 빌드 테스트 진행

**자동화 가능성**: ⭐⭐⭐⭐⭐ (매우 높음)
- ESLint/Prettier 규칙으로 일부 자동화
- 성능 메트릭 자동 측정
- 애니메이션 복잡도 분석

### 2. **컴포넌트 분리** 패턴  
**빈도**: 중간
**단계**:
1. 단일 파일 크기 증가 감지
2. 논리적 단위로 분리 계획
3. 파일별 책임 분리 실행
4. import/export 정리

**예시**:
- page.tsx 1,665줄 → 10개 컴포넌트로 분할 제안

**자동화 가능성**: ⭐⭐⭐ (보통)
- AST 분석으로 분리 후보 감지
- 컴포넌트 추출 스크립트 생성

### 3. **스킬 개발** 패턴
**빈도**: 낮음 (하지만 중요)
**단계**:
1. 반복 작업 패턴 감지
2. 자동화 가능성 평가  
3. 스킬 구조 설계
4. 스크립트 개발
5. 테스트 및 문서화

**예시**:
- 코드 품질 검사 자동화 스킬 개발 시작

**자동화 가능성**: ⭐⭐ (낮음 - 창의성 필요)

---

## 📈 패턴 분석 인사이트

### 높은 자동화 가치
1. **성능 검사**: 애니메이션 복잡도, 번들 크기
2. **코드 품질**: ESLint, TypeScript 에러
3. **빌드 검증**: 성공/실패, 경고 메시지

### 개선 제안
1. **실시간 피드백**: Critic → Developer 통신 자동화
2. **메트릭 추적**: 성능 변화 히스토리 관리
3. **패턴 학습**: 반복되는 개선사항 데이터베이스화
