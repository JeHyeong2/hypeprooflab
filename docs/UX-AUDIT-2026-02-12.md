# HypeProof Lab UX Audit Report

**Date:** 2026-02-12  
**Auditor:** AI UX Audit Agent  
**Site:** https://hypeproof-ai.xyz  
**Stack:** Next.js 16.1.6, React 19.2.3, Framer Motion 12.33, Tailwind CSS  

---

## Executive Summary

HypeProof Lab은 비주얼과 인터랙션 디자인은 세련되나, **SEO/SSR 치명적 결함**, **누락된 필수 에셋**, **네비게이션 불일치**, **중복 코드** 등 구조적 문제가 다수 발견됨. 특히 홈과 칼럼 목록 페이지가 클라이언트 사이드 렌더링이라 크롤러에 콘텐츠가 노출되지 않는 점은 즉시 수정 필요.

---

## Critical 🔴

### C1. 필수 에셋 파일 누락 (favicon, manifest, apple-touch-icon)
- **현상:** `favicon.ico`, `apple-touch-icon.png`, `manifest.json`, `og-image-square.png` 파일이 `public/`에 존재하지 않음
- **영향:** 브라우저 탭 아이콘 미표시, PWA 동작 불가, 소셜 공유 시 OG 이미지 깨짐
- **layout.tsx**에서 이 파일들을 모두 참조하고 있으므로 404 에러 다수 발생
- **개선:** 해당 에셋 파일들 생성 및 배치
- **작업량:** S

### C2. 홈/칼럼 목록 SSR 콘텐츠 미노출 — SEO 치명적
- **현상:** 홈(`/`)과 칼럼 목록(`/columns`)이 `'use client'` HomeClient/ColumnsListClient로 전체 렌더링되어, 크롤러가 본문 콘텐츠를 전혀 읽지 못함
- **web_fetch 결과:** 홈 → "H / We don't chase Hype. We Prove it." 만 추출됨. 칼럼 목록 → 동일. 실제 칼럼 리스트, 기능 소개, 팀 소개 등 모든 콘텐츠가 JS 실행 후에만 렌더링
- **영향:** Google 등 검색엔진 인덱싱 불가, 소셜 미리보기 빈 페이지
- **개선:** 서버 컴포넌트에서 정적 HTML 생성 후 클라이언트 하이드레이션. HomeClient의 구조를 서버/클라이언트 분리
- **작업량:** L

### C3. 칼럼 상세 페이지 본문 SSR 미노출
- **현상:** `/columns/2026-02-12-electric-girl` 크롤러 접근 시 본문 콘텐츠 안 보임 (ColumnArticle이 `'use client'`)
- **page.tsx는 서버 컴포넌트**지만 모든 렌더링을 `ColumnArticle` 클라이언트 컴포넌트에 위임
- **개선:** article 본문과 메타데이터는 서버에서 렌더, 인터랙티브 요소(좋아요, 댓글)만 클라이언트
- **작업량:** M

## High 🟠

### H1. 네비게이션 컴포넌트 3중 중복
- **현상:** 네비게이션 바가 3곳에서 각각 하드코딩됨:
  1. `Navigation.tsx` (홈 전용, framer-motion 포함)
  2. `ColumnsListClient.tsx` 내 인라인 nav
  3. `novels/page.tsx` 내 인라인 nav
  4. `my-activity/MyActivityClient.tsx` 내 인라인 nav
- **문제:** 링크, 스타일, 기능이 페이지마다 다름. 홈 nav에는 Auth버튼·모바일메뉴 있지만, novels nav에는 없음
- **영향:** 유지보수 어려움, 사용자 혼란 (네비 구조가 페이지마다 다름)
- **개선:** 공통 Navigation 컴포넌트 하나로 통일
- **작업량:** M

### H2. 모바일 메뉴에서 내부 페이지 링크 누락
- **현상:** MobileMenu에 "What We Do", "Team" (홈 섹션 앵커), "Columns", "Novels", "Contact"만 있음
- **누락:** "내 활동(My Activity)" 링크 없음. 로그인한 사용자가 모바일에서 내 활동 페이지에 접근 불가
- **개선:** 모바일 메뉴에 My Activity, AuthButton 추가
- **작업량:** S

### H3. 홈 페이지 칼럼 데이터 하드코딩
- **현상:** `HomeClient.tsx`에 `columnsData` 객체로 칼럼 3개가 하드코딩됨
- **문제:** 새 칼럼 발행해도 홈의 "최신 칼럼" 섹션이 자동 갱신 안 됨. CLAUDE.md에서도 하드코딩 금지 원칙
- **개선:** 서버 컴포넌트에서 `getAllColumns()` 호출 후 최신 3개를 HomeClient에 props로 전달
- **작업량:** S

### H4. Google Search Console verification 코드가 placeholder
- **현상:** `layout.tsx`에 `google: "your-google-verification-code"`, `yandex: "your-yandex-verification-code"` 그대로 남아 있음
- **영향:** 검색엔진 소유권 인증 미완료
- **개선:** 실제 인증 코드로 교체하거나, 설정 전이면 해당 필드 삭제
- **작업량:** S

### H5. `my-activity` 페이지 메타데이터 한국어 고정
- **현상:** `title: '내 활동 | HypeProof AI'`로 한국어만 지원
- **영향:** 영어 사용자에게 한국어 title 노출
- **개선:** locale에 따른 동적 title 또는 영어 병기
- **작업량:** S

## Medium 🟡

### M1. `<html lang="ko">` 고정 — 언어 전환 미반영
- **현상:** layout.tsx에서 `<html lang="ko">`로 고정. 영어 전환해도 변경 안 됨
- **영향:** 접근성(스크린리더), SEO(hreflang 신호 부정확)
- **개선:** locale 상태를 서버까지 전달하거나, middleware에서 lang 속성 동적 설정
- **작업량:** M

### M2. 홈 페이지 `#community` 앵커 — 데스크톱 NavLink에 있지만 모바일 메뉴에 없음
- **현상:** 데스크톱 nav에는 "Community" 링크가 있지만 모바일 메뉴에는 없음
- **개선:** 모바일 메뉴에 Community 추가
- **작업량:** S

### M3. `img` 태그 직접 사용 (HomeClient NovelsPreview)
- **현상:** `<img src="/authors/cipher.png">` 사용 — Next.js `Image` 컴포넌트 미사용
- **영향:** 이미지 최적화(WebP 변환, lazy loading, 사이즈 최적화) 미적용
- **개선:** `next/image` Image 컴포넌트로 교체
- **작업량:** S

### M4. ColumnArticle — `rehype-sanitize`로 인한 콘텐츠 손실 가능성
- **현상:** ReactMarkdown에 `rehypeSanitize` 적용. 만약 칼럼에 iframe, custom HTML, 이미지 속성 등이 있으면 제거됨
- **개선:** 커스텀 sanitize 스키마로 필요한 태그 허용
- **작업량:** S

### M5. 소설 "읽기 시작" 버튼이 최신 챕터로 연결
- **현상:** novels/page.tsx에서 `latestChapter` 슬러그로 "읽기 시작" 링크 생성. 신규 독자는 최신화가 아닌 1화부터 읽어야 함
- **개선:** "1화부터 읽기" + "최신화 읽기" 두 개 CTA 제공
- **작업량:** S

### M6. SkeletonLoader/LoadingSpinner — 사용처 없음
- **현상:** `SkeletonLoader.tsx`와 `LoadingSpinner.tsx`에 다양한 스켈레톤/로딩 컴포넌트가 정의되어 있으나, 실제 페이지에서 사용하지 않음 (grep 결과 import 없음)
- **영향:** 번들 사이즈 낭비는 아니나 (tree-shaking), 정작 필요한 곳(칼럼 목록 로딩 등)에 스켈레톤이 없음
- **개선:** 칼럼/소설 목록 로딩 시 `ColumnCardSkeleton` 활용
- **작업량:** S

### M7. CursorFollower — 모바일에서 불필요한 렌더링
- **현상:** HomeClient에서 `<CursorFollower />`가 데스크톱/모바일 구분 없이 렌더링
- **영향:** 모바일에서 마우스 커서 효과 무의미, 불필요한 이벤트 리스너
- **개선:** `matchMedia` 또는 CSS breakpoint로 데스크톱만 렌더링
- **작업량:** S

### M8. Hero.tsx 내 조건부 `useTransform` 호출 — React Hook 규칙 위반
- **현상:** `Hero.tsx`에서 `animationConfig.enableParallax` 조건에 따라 `useTransform` 인자가 달라짐. 이 자체는 hook 규칙 위반은 아니지만, 코드 가독성과 유지보수가 어려움
- **개선:** useTransform은 항상 호출하되, 결과값 사용 여부를 조건 분기
- **작업량:** S

## Low 🟢

### L1. Discord 링크 `https://discord.gg/hypeproof` — 유효성 미확인
- **현상:** 여러 곳에서 사용되는 Discord 링크가 실제로 유효한지 확인 필요
- **작업량:** S

### L2. Footer "Coming Soon" 항목 다수
- **현상:** Twitter, LinkedIn, Events, Newsletter 모두 "Coming Soon"
- **영향:** 사이트가 미완성으로 보일 수 있음
- **개선:** 준비 안 된 항목은 숨기거나, 예정 일정 표시
- **작업량:** S

### L3. `auth/error` 페이지 — 네비게이션/헤더 없음
- **현상:** 에러 페이지에 사이트 로고나 nav가 없음. "홈으로 돌아가기" 버튼만 존재
- **개선:** 최소한의 로고 + 홈 링크 헤더 추가
- **작업량:** S

### L4. `my-activity` 활동 목록에 slug만 표시
- **현상:** 좋아요/북마크 목록에서 칼럼 제목 대신 `content_slug` (예: `2026-02-12-electric-girl`) 그대로 표시
- **영향:** 사용자가 어떤 글인지 알기 어려움
- **개선:** slug로 칼럼 메타데이터 조회하여 제목 표시
- **작업량:** M

### L5. `columns/page.tsx` 메타데이터 영어 고정
- **현상:** `title: 'Columns | HypeProof AI'`, `description`도 영어만
- **개선:** novels처럼 locale별 동적 메타데이터
- **작업량:** S

### L6. SearchAction schema에 `/search` 페이지 없음
- **현상:** layout.tsx structured data에 SearchAction → `https://hypeproof-ai.xyz/search?q=...` 정의했으나, 실제 검색 페이지 미구현
- **개선:** SearchAction 스키마 제거하거나 검색 기능 구현
- **작업량:** S (제거) / L (구현)

### L7. HeroSection.tsx와 Hero.tsx 분리 — 네이밍 혼란
- **현상:** `HeroSection.tsx` (279줄)에 `FloatingOrb` 등 유틸리티 export, `Hero.tsx` (212줄)에 실제 Hero 컴포넌트. 둘의 역할 구분이 이름만으로 불명확
- **개선:** HeroSection → HeroEffects 등으로 리네이밍
- **작업량:** S

### L8. `<a>` vs `<Link>` 혼용
- **현상:** Footer에서 내부 링크(`/columns`, `/novels`)를 `<a>` 태그로 사용 — Next.js `<Link>` 미사용
- **영향:** 클라이언트 사이드 네비게이션 불가, full page reload 발생
- **개선:** 내부 경로는 `next/link` 사용
- **작업량:** S
