## JY (신진용) — 교육용 실행환경 구축

**역할**: 유료 파일럿이 실제로 돌아갈 수 있는 교육용 실행환경 (OAuth 기반) 구축
**서포트**: Jay ↔ JY (Product 구조/요구사항 ↔ 가격/라이선스 기술 구현)

**Objective**: 유료 파일럿 1회가 수강생 기준으로 문제없이 운영될 수 있도록, 로그인부터 실습까지 이어지는 교육용 실행환경 구축

| KR | 내용 | 팀 KR |
|----|------|-------|
| KR1 | 수강생 5명+ Google OAuth 로그인 → 실습 환경 진입 → 1개 과제 실행 완료 | KR1, KR2 |
| KR2 | 파일럿 1회 운영 중 치명적 장애(진행 중단) 0건 | KR1 |
| KR3 | 운영자 가이드 문서 1종 + 제3자가 문서만으로 환경 세팅 및 실행 성공 | KR2 |

**산출물**: 수강생 로그인/실습 로그, 파일럿 운영 기록, 실행환경 세팅 가이드

**역할**: 동아일보 파일럿을 '구매로 이어지는 레퍼런스'로 설계 및 완성
**서포트**: Ryan → 문두열 (wow 포인트 경험 설계), TJ → Ryan (콘텐츠화), Kiwon → Ryan (GTM 연결)

**Objective**: 동아일보 파일럿에서 참가자가 스스로 미래 역할에 대한 답을 도출하는 경험을 설계하고, 외부 고객이 구매 의사를 가지는 레퍼런스 생성

| KR | 내용 | 팀 KR |
|----|------|-------|
| KR1 | wow 포인트(역할 재정의 순간)가 리뷰/피드백에 명시적 언급 2건+ | KR3 |
| KR2 | 외부 공개 가능한 case study (Notion) 1종 완성 | KR3 |
| KR3 | 동아일보 후속 제안 미팅 1회+ 발생 | KR3 |

**산출물**: Case study, wow 포인트 설계 문서, 설문 설계 문서, 동아일보 후속 제안서/미팅 기록

**역할**: 팀의 실행 결과를 외부 매출로 연결하는 '레퍼런스 콘텐츠 자산 시스템' 오너
**서포트**: TJ → Ryan (레퍼런스 → 전달 가능 형태), TJ → Kiwon (세일즈/GTM 콘텐츠)
Ryan → TJ (핵심 포인트), Kiwon → TJ (타겟 정제), JY → TJ (데모/기술)

| KR | 내용 | 팀 KR |
|----|------|-------|
| KR1 | 치과 파일럿 세일즈용 콘텐츠 패키지 1종 (영상 + 사례 + B/A + 1p 요약) | KR1, KR3 |
| KR2 | 콘텐츠가 실제 세일즈에서 3회+ 활용 | KR3 |
| KR3 | 콘텐츠 기반 잠재 고객 반응 1건+ (문의/미팅/파일럿 논의) | KR3 |

**산출물**: 레퍼런스 영상 1개, 세일즈 콘텐츠 패키지, 활용 기록, 유입/문의 사례

**서포트**: BH → Jay/Ryan/Kiwon (교육 구조/레퍼런스/메시지 연결)
