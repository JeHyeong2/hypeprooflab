## (a) 케이스 개요

### C02 정의

C02는 **동아사이언스**와 협력하여 과학동아AiR 구독자 기반 학생·학부모를 대상으로 운영하는 번들 워크숍이다. 초등 4-6학년 + 학부모 B2C 워크숍(동아미디어센터 직영)인 C01과 달리, C02는 영재원/과학고 준비생을 타겟으로 한 **B2C 유료 번들 워크숍**으로 독립 수익을 창출하면서 동아사이언스의 AiR 구독 유지율 KPI를 동시에 달성하는 구조다.

## (c) 커리큘럼 적응 (STEM Edition)

### 적응 원칙

- 4파트 구조(Mindset → Collaboration → Directing → Achievement)는 **반드시 유지**한다.
- 전체 240분(4시간) 분량을 유지한다.
- 어휘와 예시를 STEM 도메인으로 전환: "게임 만들기" → "과학 문제 시뮬레이터 만들기"
- 학부모의 역할은 Integrator 그대로 유지하되, 현실화 질문이 "공학적 제약" 중심으로 변경된다.
- 영재원/과학고 준비생(중1~고1 수준)에 맞게 난이도 조정: 추상적 사고 가능, 논리적 지시 훈련에 집중.

### Part 1: 관점의 전환 — STEM Mindset (40분, 구조 변경 없음)

| 항목 | 기본 커리큘럼 | STEM Edition 변형 |
|------|-------------|-----------------|
| 오프닝 키 메시지 | "코딩하는 사람이 아닌 AI를 지휘하는 설계자" | "실험을 설계하는 과학자처럼 — AI에게 무엇을 시킬지 결정하는 사람" |
| AI 라이브 데모 | AI로 30초 게임 생성 | AI로 30초 물리 시뮬레이터(공 굴리기, 중력 변수 조정) 생성 시연 |
| 핵심 토론 | "벽돌 나르는 사람 vs 설계도면 그리는 사람" | "실험을 반복하는 조교 vs 가설을 설계하는 과학자" |
| 역할 카드 | Visionary(아이) / Integrator(부모) | Scientist(아이: 가설 설계자) / Engineer(부모: 제약 검토자) |
| 팀 이름 정하기 | 자유 | 과학 요소 권장 (예: "중력팀", "양자팀", "파동팀") |

**시간 조정**: 변경 없음 (40분)

### Part 2: 협업과 조율 — Problem Design Workshop (50분, 핵심 변형)

| 항목 | 기본 커리큘럼 | STEM Edition 변형 |
|------|-------------|-----------------|
| 샘플 체험 | 데모 게임 2~3개 플레이 | AI가 만든 과학 시뮬레이터 2개 체험 + "이 시뮬레이터는 어떤 과학 개념을 표현하나?" 분석 |
| 브레인스토밍 | 아이디어 발산 (자유 주제) | **과학 문제 정의**: "어떤 과학 현상을 친구들에게 설명하고 싶어?" 포스트잇 발산 |
| 기획서(PRD) | 게임 기획서: 이름, 캐릭터, 규칙, 승리 조건 | **STEM 기획서**: 설명하려는 과학 개념, 사용자 인터랙션 방식, 시각화 방법, 검증 기준 |
| 부모 역할 가이드 | "그럼 어떻게 이기는 거야?" | "이 현상은 어떤 조건에서 달라져?" (공학적 제약 질문) |
| 휴식 | 10분 | 10분 (변경 없음) |

**STEM 기획서 필수 항목**:
1. 시뮬레이터/앱 이름
2. 설명하려는 과학 개념 (예: 중력 가속도, 파동, 화학 반응)
3. 사용자가 조작하는 변수 (예: 질량, 속도, 온도)
4. 출력 결과 (예: 그래프, 애니메이션, 숫자 변화)
5. "왜 이게 흥미로운가?" 한 줄 설명

**시간 조정**: 변경 없음 (50분). 과학 개념 선정 시간이 자연스럽게 브레인스토밍에 통합됨.

### Part 3: 지휘와 소통 — AI-Directed Science Build (90분, 핵심 변형)

| 항목 | 기본 커리큘럼 | STEM Edition 변형 |
|------|-------------|-----------------|
| 맥락 설계 훈련 | "그냥 해줘" vs 구체적 지시 비교 | "과학 개념 없이 지시" vs "개념 + 조건 + 출력 형식 포함 지시" 비교 체험 |
| 실습 1차 | 게임 골격 완성 | 시뮬레이터/앱 골격 완성: AI에게 과학 개념 + 인터랙션 구현 지시 |
| 10초 피드백 루프 | 지시 → 결과 확인 → 수정 → 반복 | "지시 → 결과 확인 → 과학적으로 맞나? → 수정" 루프 (과학적 정확성 검토 추가) |
| 실습 2차 | 기능 추가, 디자인, 버그 수정 | 변수 추가, 시각화 개선, "포기의 미덕" 훈련 |
| 중간 점검 | 진행 상황 공유 | 각 팀 "지금 어떤 과학 개념을 구현 중?" 30초 발표 (과학 개념 인식 강화) |

**STEM 특화 조교 역할**:
- 과학적 오류 발생 시 "AI가 틀린 것 같아요. 실제 물리 법칙과 비교해볼까요?" 유도
- 추가 도전 과제: "중력을 달에서처럼 1/6로 바꾸면 어떻게 될까? AI에게 시켜봐"
- 너무 복잡한 시도 시 "핵심 변수 하나만 먼저 구현하고, 나머지는 다음에"

**시간 조정**: 변경 없음 (90분). 중간 점검에 30초 발표 추가 → 실습 2차를 5분 단축하여 흡수.

### Part 4: 성취와 비전 — STEM Achievement (60분, 발표 형식 변형)

| 항목 | 기본 커리큘럼 | STEM Edition 변형 |

| 시상 카테고리 | 창의성, 완성도, 발표, 협업, AI 마스터 | 동일 5개 유지. 강사 재량으로 과학 정확도상 추가 가능 |
