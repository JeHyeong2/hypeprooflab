## 1. 학교 IT 인프라 점검 체크리스트 (D-14 실시)

### 1.1 점검 방법
- Filamentree 조교 1명이 학교 정보담당교사와 함께 컴퓨터실 방문 (약 1시간)
- PC 3대를 랜덤 샘플로 선택하여 전 항목 테스트
- 결과를 IT 점검 보고서로 작성하여 Jay에게 D-13까지 제출

### 1.2 필수 점검 항목

| # | 항목 | 최소 요건 | 테스트 방법 | 미달 시 대응 |
|---|------|---------|-----------|------------|
| 1 | **PC 대수** | 25대 이상 (학급 인원 이상) | 컴퓨터실 비치 대수 실측 | 부족 시 2~3인 1PC 공유 (팀 구성 조정) |
| 2 | **OS 버전** | Windows 10 이상 또는 Chrome OS | 시작 메뉴 → 설정 → 정보 | Windows 7 이하: 해당 PC 사용 불가, 학생 재배치 |
| 3 | **Chrome 버전** | Chrome 90 이상 | 주소창에 `chrome://version` | 구버전: Chrome 업데이트 요청 (정보담당교사) |
| 4 | **모니터 해상도** | 1280×720 이상 | 바탕화면 우클릭 → 디스플레이 설정 | 1024×768: AI 도구 UI 일부 잘림, 조교가 개별 지원 |
| 5 | **AI 서비스 접속** | Claude.ai, ChatGPT 접속 가능 | 각 URL 직접 접속 테스트 | **차단 시: 아래 '나이스망 대응' 절차 실행** |
| 6 | **Antigravity/Cursor 접속** | 코딩 도구 웹 버전 접속 가능 | URL 접속 테스트 | 차단 시: 대안 도구 또는 핫스팟 전환 |
| 7 | **네트워크 속도** | 다운로드 50Mbps 이상 | fast.com 접속 | 미달: 핫스팟 병행, AI API 호출 빈도 조절 |
| 8 | **동시 접속 안정성** | 25대 동시 AI 서비스 접속 | 가능하면 방과후 시간에 25대 동시 테스트 | 불안정: 팀별 순차 접속 전략 (5팀씩 시간차) |
| 9 | **USB 포트** | PC당 USB-A 1개 이상 | 육안 확인 | 없으면 오프라인 백업 파일을 네트워크 공유 폴더로 배포 |
| 10 | **프로젝터/대형 TV** | 1대 이상, HDMI 또는 VGA 입력 | 강사 노트북 연결 테스트 | 미보유: 강사 노트북 화면을 학생 PC로 화면 공유 (비효율적이나 가능) |
| 11 | **마이크/스피커** | 마이크 1개 이상 (30명 대상 발성) | 있으면 테스트, 없으면 기록 | 마이크 없음: 강사 발성으로 진행 (컴퓨터실 면적 작으면 가능) |
| 12 | **전원 콘센트** | PC당 1구 (이미 배선된 경우 확인만) | 육안 확인 | 배선 불량 PC 사용 불가 표시 |

### 1.3 나이스망 AI 서비스 차단 대응 절차

**나이스망(NEIS Network)**은 교육부가 운영하는 교육 전용 네트워크로, 외부 서비스 접속이 화이트리스트 방식으로 관리된다.

| 단계 | 행동 | 담당 | 소요 시간 |
|------|------|------|----------|
| 1 | D-14 점검에서 차단 확인 | 조교 | 즉시 |
| 2 | 정보담당교사에게 임시 해제 요청 — "AI 교육 프로그램 운영을 위해 아래 URL 임시 허용 요청합니다" + URL 목록 전달 | Jay → 정보담당교사 | D-13 |
| 3 | 정보담당교사가 학교 네트워크 관리 시스템에서 해당 URL 임시 허용 설정 | 정보담당교사 | D-10까지 |
| 4 | D-7 재접속 테스트 확인 | 조교 (원격 또는 교사 대행) | D-7 |
| 5 | 해제 불가 시 (교육청 정책상 학교 단위 해제 불가) → **핫스팟 전환** | Jay | D-7 결정 |

**임시 허용 요청 URL 목록:**
```
https://claude.ai
https://api.anthropic.com
https://chat.openai.com
https://api.openai.com
https://antigravity.dev (또는 사용 도구 URL)
https://cursor.sh
https://replit.com
```

**핫스팟 전환 시 장비:**
- Filamentree 모바일 핫스팟 6대 반입 (팀 2팀당 1대 공유)
- LTE 데이터 팩: 6대 × 10GB = 60GB (회당 ₩30,000~50,000, 비상 예비비에서 집행)
- 학교 Wi-Fi 대신 핫스팟 SSID로 전환 안내 카드 제작 (A5 1장, 핫스팟 이름/비밀번호 기재)

## 3. 오프라인 폴백 계획 (USB 기반)

### 3.1 시나리오: AI 서비스 완전 차단 + 핫스팟 미작동

| 단계 | 행동 | 소요 |
|------|------|------|
| 1 | 강사: "AI 서비스에 접속이 안 되네요! 괜찮습니다, 준비된 게 있어요" | 30초 |
| 2 | 조교: USB를 팀별 PC에 꽂고 오프라인 템플릿 폴더 열기 | 3분 |
| 3 | 강사: 템플릿 구조 설명 — "이미 기본 게임이 만들어져 있어요. 여러분은 캐릭터, 규칙, 점수를 바꾸면 됩니다" | 5분 |
| 4 | 학생: 조교가 해당 PC에서 Windows 메모장 실행. 조교가 변수 위치를 노란 형광펜으로 출력된 가이드 카드에 표시하며 안내. 학생이 변수값 수정 (캐릭터 이름, 속도, 점수) | 40분 |
| 5 | 결과물: AI 없이도 "내가 수정한 게임" 완성 — 완성도 30% 수준이지만 작동함 | |

### 3.2 USB 템플릿 구성

```
USB/
├── README.txt          ← "이 파일을 먼저 읽어주세요" (강사·조교용)
├── template-1-dodge/   ← 피하기 게임 템플릿
│   ├── index.html      ← 브라우저에서 열면 바로 실행
│   └── guide.txt       ← 수정 가능 변수 목록 (한글)
├── template-2-shooter/ ← 슈팅 게임 템플릿
│   ├── index.html
│   └── guide.txt
└── template-3-catch/   ← 잡기 게임 템플릿
    ├── index.html
    └── guide.txt
```

## 4. 담임교사 협력 요건

### 4.1 교사에게 요청하는 것 (최소한)

| 시점 | 요청 사항 | 교사 부담 | 대안 |
|------|----------|----------|------|
| D-10 | 사전 미팅 30분 (온라인) — 프로그램 설명, 사전 과제 가이드 전달 | 낮음 | 가이드 PDF만 이메일 전달 (미팅 생략 가능) |
| D-1~D-Day 1교시 | 사전 과제 진행 40분 — 팀 구성 + 기획서 작성 | 중간 | **교사 미진행 시: 워크숍 첫 15분에 강사가 기획서 작성 진행 (Part 1을 5분으로 추가 축소)** |
| D-Day | 교실 상주, 학생 질서 관리 | 낮음 | 필수 — 교사 부재 시 학생 통제 어려움 |
| D-Day Part 3 | 팀별 순회하며 간단한 질문 ("뭐가 다르게 나왔어?") | 중간 | 조교가 대행 가능 (교사 순회 없어도 운영 가능) |
| D+1 | 교사 설문 5문항 응답 (3분) | 매우 낮음 | QR코드로 즉시 응답 |

### 4.2 교사에게 요청하지 않는 것 (명확히)

- AI 도구 사용법 교육 → 강사·조교 전담
- 기술적 트러블슈팅 → 조교 + 정보담당교사
- 커리큘럼 수정·판단 → 강사 전권
- 추가 수업 준비 → 가이드 제공, 준비 불필요
- 성적 반영 → 창의적 체험활동 참여로 인정 (성적과 무관)

### 4.3 교사 비협조 시 대응 매트릭스

| 비협조 유형 | 영향 | 대응 |
|-----------|------|------|
| 사전 과제 미진행 | 높음 | 워크숍 첫 15분을 기획서 작성으로 전환. Part 1 AI 데모 5분으로 축소. Part 3-B 35분으로 축소 |
| 워크숍 중 교실 이탈 | 중간 | 강사가 교실 질서 관리 겸임. 조교 1명을 질서 담당으로 전환 (실습 지원 2명으로 감소) |
| 사전 미팅 불응 | 낮음 | 가이드 PDF 이메일 전달로 대체. D-Day 아침 10분 브리핑으로 보완 |
| 프로그램 자체 부정적 | 높음 | 학교장 경유하여 교사 설득. 불가 시 해당 학교 포기 (다른 학교로 대체) |

## 5. 학생 그룹 전략 (Child-Only Mode)

### 5.1 팀 구성 원칙

| 항목 | 기준 |
|------|------|
| 팀 크기 | 2~3명/팀 (이상적: 3명) |
| 팀 수 | 25~30명 기준 → 10~12팀 |
| 구성 방법 | 담임교사가 사전 과제 시 구성 (짝꿍 기반 또는 자유 선택) |
| 역할 분담 | 팀 내에서 자연스럽게 — 강제 배정하지 않음 |

### 5.2 팀 내 역할 가이드 (강사가 Part 2에서 안내)

| 역할 | 설명 | 실제 행동 |
|------|------|----------|
| **아이디어맨** | 뭘 만들지 결정 | "우리 게임은 이렇게 하자!" |
| **타이핑맨** | AI에게 지시 입력 | 키보드로 프롬프트 타이핑 |
| **체크맨** | 결과 확인 + 피드백 | "여기 캐릭터가 안 움직여!" → 수정 요청 |

> 역할은 고정이 아님 — "역할 교대도 좋아요! 다 해보세요" 안내

### 5.3 학부모 참여 시 (희망 학부모 5~10명)

| 상황 | 대응 |
|------|------|
| 학부모 5명 이하 참관 | 학부모는 자녀 팀에 합류 → 해당 팀은 부모-자녀 Co-Founding 모드 (원본 커리큘럼과 동일) |
| 학부모 10명 이상 참관 | 학부모 별도 좌석 배치, 참관만 (팀 활동 미참여) — 인원이 많으면 동선 혼잡 |
