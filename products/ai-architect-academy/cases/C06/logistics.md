## 1. 장소 요건

### 서울 (동아미디어센터)

| 항목 | 요건 | 확인 방법 |
|------|------|----------|
| **수용 인원** | 40~50명 (20팀 × 2명 + 스태프 8명) | 평면도 확인 |
| **좌석 배치** | 팀별 테이블 (2인 1팀) × 20개. 아일랜드형 배치 권장 (조교 동선 확보) | 사전 방문 세팅 확인 |
| **전원 콘센트** | 팀당 최소 2구 이상. 멀티탭 10개 자체 반입으로 보완 | 전기 도면 확인 |
| **Wi-Fi** | 동시 접속 60대 이상 지원. 다운로드 50Mbps+ | 속도 테스트 (D-7) |
| **프로젝터/스크린** | 200인치 이상 스크린 또는 75인치 이상 대형 모니터 × 2 | 장비 확인 |
| **마이크** | 무선 핀마이크 2개 (강사용·발표용) | 음향 테스트 (D-1) |
| **화장실** | 장소 내 또는 인접 (50m 이내) | 위치 확인 |
| **접근성** | 엘리베이터 있음. 지하철역 도보 10분 이내 | 사전 확인 |
| **주차** | 최소 10대 (가족 단위 방문 고려) | 주차 안내 사전 공지 |

## 5. 비상 대응 계획

### 비상 상황 1: AI 서비스 전면 장애 (Cursor + Replit 동시 다운)

**발생 가능성**: 낮음 (5% 미만). 과거 사례: 2024년 OpenAI API 다운 2회.

**즉시 조치 순서**:
1. 강사가 멘트: "AI 서비스에 일시적 문제가 있어요. 실제 개발자들도 이런 상황에서 계획을 세우죠! Part 2로 잠시 돌아갑니다."
2. 조교: USB 오프라인 템플릿 즉시 배포 (어린이동아 소재 기반 HTML 게임 스켈레톤)
3. 오프라인 템플릿으로 기본 게임 수정 작업 진행 (CSS·텍스트 변경 레벨)
4. 서비스 복구 시 즉시 AI 작업 재개
5. 복구 불가 시: Part 4를 Part 3 바로 뒤로 당기고, 발표 시간 확대 (기획서 발표 포맷으로 전환)

### 비상 상황 2: 참가팀 수 급감 (당일 5팀 이상 노쇼)

**발생 가능성**: 낮음 (가족 단위 노쇼는 통상 5~10%).

**즉시 조치 순서**:
1. D-1 문자 발송 후 응답 없는 팀에 당일 오전 전화 확인
2. 15팀 이하 등록 확인 시 조교 1명 조기 귀가 (원가 절감)
3. 좌석 재배치 — 12팀 이하 시 반원형 또는 U자형으로 변경 (친밀감 강화)
4. 발표 시간 팀당 5분으로 확대 (Part 4 여유 활용)

### 비상 상황 3: 구독자 확인 불가 (CRM 시스템 오류)

**발생 가능성**: 중간 (10~15%, 새로운 데이터 연동 시 초기 오류 가능).

**즉시 조치 순서**:
1. 접수 시 수기 구독번호 확인 리스트 비교 (사전 준비 필수)
2. 확인 불가 시 "구독자 선언 확인서" 현장 서명 수령 후 구독자 가격 적용
3. 사후 동아이지에듀와 대조 확인, 이의 없을 시 처리 완료
4. 시스템 오류 재발 방지: 동아이지에듀에 D-3 기준 오프라인 구독자 명단 PDF 수령 요청 (이후 표준 절차로 도입)
