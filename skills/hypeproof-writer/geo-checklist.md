# GEO 최적화 체크리스트

> Generative Engine Optimization — AI 검색 엔진 인용 최적화
> HypeProof Lab 콘텐츠 제출 전 필수 확인 사항

---

## 빠른 채점 (제출 전 자가 평가)

```
python3 skills/hypeproof-writer/tools/geo_qa_local.py [파일경로]
```

목표: **70점 이상** (제출 가능) / **90점 이상** (Fast-track)

---

## 1. 인라인 인용 (25점) — 최우선 항목

**효과**: AI 인용 확률 30-40% 향상

### 체크리스트

- [ ] **최소 3개** 인라인 인용 포함 (감점 방지)
- [ ] **5개 이상** 인라인 인용 포함 (만점 25/25)
- [ ] 각 인용에 출처 URL 또는 저자/기관명 포함
- [ ] 학술 논문, 공식 보고서, 신뢰 기관 우선
- [ ] 인용이 주장과 직접 관련됨

### 인라인 인용 형식

```markdown
<!-- 형식 1: 링크 포함 -->
[Stanford HAI 2025 보고서](https://hai.stanford.edu/report)에 따르면...

<!-- 형식 2: 저자 언급 -->
McKinsey & Company(2025)의 분석에 따르면...

<!-- 형식 3: 출처 명시 -->
OpenAI의 공식 문서에 의하면...
```

### 피해야 할 것

- "연구에 따르면..." (출처 불명)
- 위키피디아 단독 인용
- 날짜가 오래된 자료 (3년 이상)

## 2. 구조 (20점) — 두 번째로 중요

**효과**: AI 인용 확률 40% 향상

### 체크리스트

- [ ] **H2 헤딩 4개 이상** (주요 섹션)
- [ ] **H3 헤딩 3개 이상** (서브섹션)
- [ ] **불릿 리스트 8개 이상** 또는 **넘버링 리스트**
- [ ] 헤딩이 질문 형태면 더 효과적 (FAQ 구조)
- [ ] 각 섹션이 독립적으로 읽힐 수 있음

### 좋은 H2 예시

```markdown
## GEO란 무엇인가?          ← 질문형 (FAQ 인용에 유리)
## 핵심 최적화 전략 5가지    ← 숫자 포함 (명확성)
## 실제 적용 사례            ← 구체성
## 자주 묻는 질문            ← FAQ 섹션 (AI 인용 최적)
```

## 4. 단어 수 (15점)

**효과**: 긴 형식 콘텐츠는 3배 더 인용됨

### 배점 기준

| 단어 수 | 점수 |
|---------|------|
| 2,000+ | 15점 (만점) |
| 1,500-1,999 | 10점 |
| 1,000-1,499 | 5점 |
| 1,000 미만 | 0점 |

### 체크리스트

- [ ] **한국어 기준 2,000자 이상** (단어 수 아닌 글자 수로 계산)
- [ ] 영어 기준 2,000 단어 이상
- [ ] 분량을 위한 반복 금지 (내용 충실도 우선)

## 5. 통계/데이터 (10점)

**효과**: 수치 포함 주장은 신뢰도 급상승

### 체크리스트

- [ ] **수치 포함 주장 4개 이상**
- [ ] 퍼센트, 배수, 금액, 기간 등 구체적 수치
- [ ] 각 수치에 출처 명시
- [ ] 최신 데이터 우선 (1-2년 이내)

### 좋은 예시

```
✅ "GPT-4는 법률 바 시험에서 90번째 백분위수를 기록했다 (OpenAI, 2023)"
✅ "인라인 인용이 있는 콘텐츠는 30-40% 더 자주 AI에 인용된다"
✅ "2024년 글로벌 AI 시장 규모는 1,840억 달러에 달한다 (Statista, 2024)"

❌ "많은 연구에서 효과가 입증되었다"
❌ "상당한 성과를 보였다"
```

## 8. 가독성/유창성 (5점)

**효과**: AI가 긴 문장을 요약할 때 오류 발생 → 인용 기피

### 체크리스트

- [ ] **평균 문장 길이 60자 이하** (한국어 기준)
- [ ] 한 문장에 하나의 핵심 아이디어
- [ ] 수동태 최소화
- [ ] 전문 용어 사용 시 즉시 설명
- [ ] 단락은 4-5문장 이하

### 자동 수정 가능

긴 문장 예시:
```
❌ "최근 AI 기술의 급속한 발전으로 인해 검색 엔진 최적화의 패러다임이 기존의 키워드 중심에서 의미론적 이해와 문맥 기반의 응답 생성 방식으로 전환되고 있는 추세에 있다."

✅ "AI 기술이 빠르게 발전하고 있다."
✅ "이에 따라 검색 최적화의 패러다임도 변화했다."
✅ "키워드 중심에서 의미 기반으로 전환된 것이다."
```

## 9. 키워드 스터핑 감지 (감점 -10점)

**효과**: 과도한 키워드 반복은 특히 Perplexity에서 역효과

### 체크리스트

- [ ] 같은 키워드가 3회 이상 연속 등장하지 않음
- [ ] 동의어와 관련 표현 활용
- [ ] 키워드 밀도 3% 이하 유지
- [ ] 자연스러운 문장 흐름 우선

### 감지 기준 (geo_qa_local.py 자동 검사)

- 특정 단어가 100단어당 5회 이상 → 경고
- 특정 단어가 100단어당 8회 이상 → 자동 감점
