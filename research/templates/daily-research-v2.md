# Daily Research Template v2

## 구조

```markdown
# OpenClaw Daily Research — YYYY-MM-DD

## [이모지] [제목] — [임팩트 부제]

[훅 문장. 1-2문장. 짧고 강렬하게 시작.]

[배경/맥락 2-3문장. 왜 이게 지금 나왔는지.]

---

### 🔥 [핵심 포인트 1]

[스토리텔링으로 풀어쓰기. 최소 3문장.]
[숫자가 있으면 비유로 맥락 부여: "1M 토큰 = 해리포터 전권"]
[비판적 코멘트 1개 이상 필수]

---

### 📊 [핵심 포인트 2]

[데이터/분석/반응 등]
[테이블은 Discord에서 깨지므로 서술형으로]
[인용문 활용 권장: "Sam Altman은 X에서 이렇게 쏘아붙였다..."]

---

### 🤔 그래서? (So What)

[실제 영향. 우리에게 의미하는 것.]
[액션 아이템 또는 시사점]
[마무리 한 문장. 여운 남기기.]

---

🔗 **Sources**
• [제목]: <URL>
• [제목]: <URL>

---

*OpenClaw Daily Research | YYYY-MM-DD*
```

## 멀티파트 시

```markdown
## [이모지] [1/N] [제목] — [이 파트의 포커스]

[위와 동일한 구조]
```

## 규칙

### 필수
- [ ] 훅 문장으로 시작 (첫 문장이 끌어당겨야 함)
- [ ] 모든 토픽에 원본 링크 필수 (`<URL>` 형식)
- [ ] 비판적 코멘트 최소 1개
- [ ] "그래서?" 섹션으로 마무리
- [ ] 파일 저장 → Discord 게시 순서 (파일 먼저!)

### 금지
- [ ] 불릿포인트 나열 (10줄 이상 금지)
- [ ] "~에 대해 알아보겠습니다" 류 표현
- [ ] "매우 흥미로운" 류 공허한 수식어
- [ ] 링크 없는 정보 인용
- [ ] 테이블 (Discord에서 깨짐)

### 권장
- "또 한 방 터뜨렸다"
- "결과가 소름 끼친다"
- "근데 함정이 있다"
- "1위긴 1위인데"
- 직접 인용: "Sam Altman은 이렇게 말했다: ..."
- 비유: "해리포터 전권 분량", "서울-부산 거리"

## 글자수 가이드

- 단일 파트: ~1800자 (Discord 2000자 한도)
- 멀티파트: 각 파트 ~1500-1800자
- 주제 1개 = ~1500자 예상
- 주제 2개 이상 = 분할 검토
