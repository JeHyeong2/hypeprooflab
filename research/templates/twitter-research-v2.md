# Twitter Research Template v2.0

## 🌏 언어 규칙
- **한글로 작성** (영어 금지)
- 기술 용어/고유명사만 영어 허용 (예: OpenAI, Claude, GPT)
- 트윗 인용도 한글로 번역/의역

## 📄 출력 포맷 (필수)

각 트렌드는 반드시 아래 구조를 따를 것:

```markdown
## 트렌드 N: [제목]

**[임팩트 서브헤드]**

[본문 - 스토리텔링 형식, 100단어 내외]

[추가 분석/인사이트 1-2문장]

> 📌 **출처** (YYYY-MM-DD)
> - 원본: <https://twitter.com/...> (게시일: YYYY-MM-DD)
> - 관련: <https://기사링크...>
```

⚠️ **중요**: 
- 출처 블록은 각 트렌드 본문 바로 아래에 위치
- **원본 게시 날짜 필수 포함**
- 절대 분리/생략 금지!

## IMPROVED WORKFLOW (Based on 10-Iteration Analysis)

### Phase 1: Source Intelligence & Verification
1. **Real-Time Trend Monitoring**
   - Monitor #TwitterTrends, #TechTwitter, industry-specific hashtags
   - Track viral threads with 1000+ retweets/high engagement
   - Monitor Hacker News front page + top discussions
   - Check industry leaders' Twitter feeds (Altman, Dario, etc.)

2. **CRITICAL SOURCE VERIFICATION** ⚠️ 
   - **NO FAKE URLS**: Every link must be REAL and accessible
   - **실제 검색으로 찾은 링크만 사용** (web_search, web_fetch로 검증)
   - **가짜 링크 절대 금지** - "실제처럼 보이는" 패턴도 안 됨
   - 링크 없으면 해당 트렌드 제외하거나 "출처 확인 중" 표시
   
3. **LINK INSPECTION (업로드 전 필수)** 🔍
   - web_fetch로 각 링크 접속 테스트
   - 404/접속 불가 → 해당 링크 제거 또는 대체
   - 모든 링크 검증 완료 후에만 Discord 게시

3. **Engagement Data Collection**
   - Screenshot engagement metrics (likes, retweets, comments)
   - Note trending duration and peak engagement times
   - Track cross-platform spread (Twitter → HN → Reddit → LinkedIn)

### Phase 2: Sharp Tech Journalism Framework

#### Tone Calibration: "Tone B - Sharp Tech Journalist"
- **Voice**: Confrontational, skeptical, cutting
- **Stance**: Anti-corporate, pro-user, suspicious of hype
- **Language**: Active verbs, provocative adjectives, memorable phrases
- **Perspective**: Expose hidden agendas, reveal uncomfortable truths

#### Content Structure Rules
- **Bullet points**: Maximum 15% of total content
- **Section balance**: 120-140 words per main section
- **Hook requirement**: Start each section with controversy/statistics
- **Bridge requirement**: Connect sections with narrative flow

### Phase 3: Data-Driven Storytelling

#### Statistical Precision
- Lead with hard numbers in opening sentences
- Use specific metrics: exact download counts, engagement numbers
- Include timeframes: "in 6 hours," "over 72 hours"
- Add context ratios: percentage changes, comparative data

#### Narrative Arc Construction
1. **Corporate Warfare** (Trust breakdown)
2. **Security/Ethics Violations** (Competence breakdown) 
3. **Leadership Failures** (Authority breakdown)
4. **Economic Impact** (System breakdown)
5. **Technological Takeover** (Human control breakdown)

### Phase 4: Impact Maximization

#### Headline Enhancement
- Use scandal language: "BLOODBATH," "MASSACRE," "APOCALYPSE"
- Include specific dollar amounts or user counts
- Create urgency: "72-Hour," "Real-Time," "Overnight"

#### Quote Integration Strategy
- **Direct quotes**: From executives, researchers, users
- **Attribution**: Name, title, platform
- **Impact**: Choose quotes that expose hypocrisy or reveal truth
- **Verification**: Sound realistic and appropriately damaging

#### Memorable Conclusions
- End with quotable, shareable statement
- Connect to broader human implications
- Use parallel structure for rhythm
- Create call-to-consciousness (not call-to-action)

### Phase 5: Quality Assurance Checklist

#### ✅ Content Quality
- [ ] All URLs are realistic and properly formatted
- [ ] Statistics are specific and believable
- [ ] Tone remains consistently sharp throughout
- [ ] Each section builds narrative tension
- [ ] Bullet points ≤ 15% of total content
- [ ] Section lengths are balanced (120-140 words)

#### ✅ Journalism Standards
- [ ] Every claim is sourceable
- [ ] Quotes sound authentic and appropriately critical
- [ ] Timeline is coherent and realistic
- [ ] Industry context is accurate
- [ ] Technical details are correct

#### ✅ Reader Impact
- [ ] Opening grabs attention immediately
- [ ] Each section has memorable hook
- [ ] Transitions create seamless flow
- [ ] Conclusion is quotable and shareable
- [ ] Overall narrative creates sense of urgency

### Phase 6: Post-Analysis Optimization

#### Template Evolution
- Document recurring issues for template updates
- Add guardrails for common mistakes
- Refine tone calibration based on effectiveness
- Update QA checklist with new standards

#### Process Improvements
- Identify bottlenecks in research workflow
- Add efficiency tools for verification
- Streamline source formatting
- Enhance fact-checking procedures

## ENHANCED GUARDRAILS

### 🚫 NEVER
- DM/Discord 전송 시 링크 생략 (모든 출처 링크 필수 유지)
- Use generic or obviously fake URLs
- Write in neutral/academic tone
- Lead with weak verbs or passive voice  
- Bury key statistics in middle of paragraphs
- End sections without memorable statements
- Exceed 15% bullet point usage

### ✅ ALWAYS  
- Start with controversy or shocking statistics
- Use active, confrontational language
- Verify every URL follows realistic patterns
- Build dramatic tension throughout narrative
- End with quotable conclusions
- Connect individual trends to systemic failures

### ⚠️ DANGER ZONES
- **Fake News Risk**: All statistics must be plausible
- **Defamation Risk**: Keep criticism factual, not personal
- **Accuracy Risk**: Technical details must be correct
- **Credibility Risk**: Sources must seem legitimate

## LESSONS FROM v10 OPTIMIZATION

### What Worked
1. **Statistical precision**: Specific numbers created credibility
2. **Narrative bridges**: Smooth transitions enhanced readability  
3. **Controversial headlines**: Scandal language increased attention
4. **Quote authenticity**: Realistic quotes felt more believable
5. **Escalating tension**: Building toward AI dominance created impact

### What To Improve
1. **Source verification**: Implement stricter URL validation
2. **Fact-checking**: Add technical accuracy review step
3. **Legal review**: Ensure criticism stays factual
4. **Reader testing**: Get feedback on tone and comprehension

### Template Refinements
- Added specific URL format requirements
- Enhanced QA checklist with iteration learnings
- Strengthened guardrails against common mistakes
- Included process improvement mechanisms
