# Karpathy: "프로그래밍이 인식 불가능하게 변했다"

## 원문 요약

Andrej Karpathy가 코딩 에이전트의 **변곡점**을 선언.

### 핵심 주장: "12월 이전과 이후는 다른 세계"

코딩 에이전트가 2025년 12월을 기점으로 실질적으로 작동하기 시작. 점진적 개선이 아니라, 모델의 품질·장기 일관성·끈기가 임계점을 넘어 대형 태스크를 밀어붙일 수 있게 됨.

### 실제 사례: DGX Spark 홈 비디오 분석

주말에 집 CCTV 분석 대시보드를 만들면서 에이전트에게 한 문장으로 지시:
- SSH 세팅 → vLLM 설치 → Qwen3-VL 벤치 → 영상 추론 서버 → 웹 UI → systemd 서비스 → 마크다운 리포트

**에이전트가 ~30분 만에 전부 완료.** 문제 만나면 스스로 검색하고, 디버깅하고, 테스트까지. 3개월 전이었으면 주말 프로젝트였을 것.

### 패러다임 전환: 코딩 → "에이전틱 엔지니어링"

> "에디터에 코드를 타이핑하던 시대는 끝났다. AI 에이전트를 띄우고, 영어로 태스크를 주고, 여러 인스턴스를 병렬로 관리하는 것이 프로그래밍이다."

가장 큰 기회는 **추상화 레이어를 올라가는 것** — 도구·메모리·지시사항을 갖춘 오케스트레이터를 세팅해서 여러 코딩 에이전트를 동시에 관리하는 구조. "agentic engineering"의 레버리지가 지금 매우 높다고.

### 한계 인정

- 하이레벨 방향 설정, 판단, 취향, 감독은 여전히 인간 몫
- 잘 정의되고 테스트 가능한 태스크에서 훨씬 잘 작동
- 태스크를 적절히 분해해서 넘기는 **직관**이 핵심 역량

### HypeProof 시각

Karpathy의 이야기는 OpenClaw 오케스트레이터(Mother) → 서브에이전트(Walter, Herald) → 병렬 태스크 구조와 정확히 일치. **코드를 짜는 능력**보다 **에이전트에게 올바른 태스크를 정의하고 분해하는 능력**이 더 중요해지는 시대.
